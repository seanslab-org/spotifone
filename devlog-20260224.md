## Fix: HID keyboard not working when audio (mic_bridge) is also running

### Problem
Audio (mic_bridge) and HID keyboard (hid_keyboard) each work individually but fail when combined. When both run together, only audio works — keyboard keys never arrive at the Mac.

### Root Causes

**Bug 1: Missing `-P input` in combined mode (`bt_init.sh:87`)**
When `KEYBOARD_FIRST=0` (normal combined mode), bluetoothd started without `-P input`. BlueZ's built-in input plugin bound L2CAP PSM 17/19 internally, preventing hid_keyboard from registering its HID profile. The flag was only present in `KEYBOARD_FIRST=1` mode — a copy-paste omission.

**Bug 2: Dual-agent conflict**
Both daemons registered their own D-Bus pairing agent and called `RequestDefaultAgent`. mic_bridge starts first and wins the slot. When hid_keyboard starts and calls `RequestDefaultAgent`, one of them loses pairing ability.

### Fixes

1. **`scripts/bt_init.sh`**: Added `-P input` to combined-mode bluetoothd startup (line 87). Updated log message.

2. **`daemon/hid_keyboard.c`**: Removed all agent registration code (~230 lines removed):
   - Removed `AGENT_PATH`, `AGENT_IFACE` constants
   - Removed `AGENT_INTROSPECT_XML`
   - Removed `agent_message_handler()` function
   - Removed `register_agent()` function
   - Removed agent vtable, object path registration, and cleanup in `main()`
   - mic_bridge's agent handles all pairing for both profiles

### Verification
- Code reviewed for correctness (no orphaned references)
- File reduced from 881 → 652 lines
- Needs on-device test: deploy, reboot, re-pair, verify both audio + keyboard work

### Lessons
- #39: `-P input` needed in ALL modes, not just keyboard-first
- #40: Only one daemon should register a BlueZ pairing agent

## Display: Vertical Boot Screen + Amlogic Boot Logo

### Goal
Replace the stock Spotify "blue house" boot logo and blank/noisy framebuffer with a branded Spotifone vertical boot screen — consistent from U-Boot through runtime.

### Architecture

Two display stages on Car Thing:

1. **Boot logo** (U-Boot) — Amlogic reads `bootup.bmp` from the logo partition (`/dev/logo`), renders to display before kernel starts. Format: 16-bit R5G6B5 BMP packed via `aml_imgpack`.
2. **Runtime framebuffer** (after bt_init.sh) — `setup_display.sh` writes raw BGR888 to `/dev/fb0`. Runs at Step 1b of bt_init (right after GPIO reset, before BT stack).

Both stages show the same visual: black background, mic icon centered on upper portion, "Spotifone" title (64pt white), "Music · Voice · Connected" tagline (28pt gray).

### Implementation

**`scripts/convert_logo.py`** — Single script generates both outputs from `logo.jpeg`:
- `logo.fb` — 480×800 BGR888 (1,152,000 bytes) for `/dev/fb0`
- `bootup.bmp` — 480×800 16-bit R5G6B5 BMP (768,066 bytes) for logo partition
- `logo.png` — preview PNG for visual verification on Mac
- Process: crops mic icon from left 33% of horizontal logo, removes white background via per-channel threshold, scales to 220px wide, composites onto black canvas, renders text with system fonts

**`scripts/setup_display.sh`** — Runtime display init:
- Unbinds ALL vtconsoles (`vtcon0`, `vtcon1`, etc.) to stop fbcon redraws
- Clears framebuffer to black with `dd if=/dev/zero`
- Writes `logo.fb` to `/dev/fb0`
- Called from `bt_init.sh` Step 1b

**`scripts/deploy.sh`** — Updated to push `logo.fb` + `setup_display.sh` to device

**Boot logo flash** — Manual process (one-time per device):
1. `python3 scripts/convert_logo.py` → generates `bootup.bmp`
2. Pull `/dev/logo` dump: `adb shell dd if=/dev/logo of=/tmp/logo.dump`
3. Unpack with `aml_imgpack.py --unpack`
4. Replace `bootup.bmp`
5. Repack with `aml_imgpack.py --pack`
6. Flash: `adb shell dd if=/tmp/logo_new.dump of=/dev/logo`

### Bug Fix: Noisy Framebuffer Background

**Problem**: Runtime framebuffer showed colored noise pixels in black areas.

**Root cause**: `setup_display.sh` tried to unbind `vtcon1` which doesn't exist on Car Thing — only `vtcon0` exists. fbcon stayed active and redrew garbage over the framebuffer after `logo.fb` was written.

**Fix**: Changed to unbind ALL vtconsoles via glob (`/sys/class/vtconsole/vtcon*/bind`), plus pre-clear framebuffer with `dd if=/dev/zero` as safety belt.
